对应的消息通知及业务规则，重置当前节点为重新提交。注意：选择【重新依次审批】，直接退回至发起人，流程状态显示为发起人名下。支持配置发起人在发起节点重新提交时关闭所有节点的自动触发，即可配置是否需要【同时触发公式，关联业务规则和第三方服务回调】。在流程审批进度、数据管理页，均可查看到当前操作退回的节点。原节点超时时间清零。从当前节点审批：退回发起人后，原节点暂停，允许发起人修改信息，原节点超时时间不中断，数据管理页看到的当前节点也是原节点。若流程节点为A => B => C ，C 将流程退回至 A ，流程将会按照 C => A => C 的链路进行流转，且不会触发对应的消息通知及业务规则，同时当前节点会保留，以便重新提交后，从当前节点继续审批。注意：选择【从当前节点审批】，退回发起人后，流程状态依然在当前审核人名下。在流程审批进度、数据管理页，均可查看到当前操作退回的节点。原节点超时时间不中断。即如果操作退回节点配置了超时处理规则，不受其影响，仍会继续计时。例如：A => B => C => D，若设置C超时30min不处理自动流转至D；当发生退回事件时，C已经用时10min，那么A节点需在20min中处理；若超过20min还未处理，则该流程将自动流转至D。收回​流程收回是指当流程到达下一审批人、执行人节点时，当前节点用户可以操作收回，重新审批或执行。说明：宜搭专属版可开启无痕收回，开启无痕收回后收回记录不在审批记录中显示，无痕收回可以在流程的全局设置中进行开启。流程收回逻辑如下：流程中的第一个节点不支持收回且不可见，可使用撤销操作。仅支持相邻节点间收回，单次收回后，不支持连续收回。如流程节点配置了节点提交规则、集成自动化触发等，不支持收回。相邻流程节点间存在自动化节点时。存在任一数据节点、子流程、卡片节点，不支持收回。仅有分支节点，支持收回。节点内配置为会签时。会签的前一个节点，仅会签节点无人操作时，支持收回。上一个节点是会签，会签内的审批人，不支持收回。多人或签时，一人收回，重新激活节点。转交、加签等方式产生的任务，支持收回。设置字段权限​流程节点的字段权限控制可以控制审批流程中某个审批节点可以查看的表单字段权限，支持根据组件名称配置可编辑、只读和隐藏三种状态。更多内容请参考流程权限控制。跳转配置​灰度中说明：本功能是专属-OA增强包专享。如您对OA增强权益版的内容感兴趣，可联系您的宜搭对接人。在业务流程中，支持「审批人」节点新增跳转配置模块，允许用户根据业务需求自定义跳转规则。跳转设置：在流程设计中，在审批人的设置中，选择“跳转设置”。添加规则：高级跳转-点击添加规则，在表格中生成一条跳转条件配置，点击某条规则。弹窗展示跳转规则配置页，需配置条件及跳转节点。⚠️注意：此处跳转节点只能选择【审批人】。设置跳转条件，如下图所示：其他配置当跳转条件均不满足时，可选择
