# 创建集成&自动化

了解了什么是集成自动化之后，你可以参考本文操作创建集成自动化。
### 从空白创建集成自动化​

你可以参考以下操作创建触发类型为表单事件触发的集成&自动化。登录宜搭工作台，单击**我的应用**。选择需要创建集成自动化的应用，进入**应用设置**页。单击集成&自动化，依次选择**新建集成&自动化** > **从空白新建**。

在弹窗内设置集成&自动化名称，选择触发类型，单击**确定**完成创建。

### 从模板创建集成&自动化​

你可以参考以下步骤从模板创建集成&自动化流。登录宜搭工作台，单击**我的应用**。选择需要创建集成自动化的应用，进入**应用设置**页。单击集成&自动化，依次选择**新建集成&自动化** > **从模板新建**。选择需要使用的模版，单击**使用**。

**Q：可以跨表单复制触发事件吗？**
只支持同一个表单内复制触发事件，
不支持跨表单复制。

**Q：简易流程在流程节点设置连接器，操作升级高级流程后连接器失效，
且复制此表单会报错：应用下包含多个连接器的审批流个数超出限制最多一个**

**Q：表单组件配置修改后，自动化中配置的条件操作符失效问题**
在集成自动化流中，可配置数据过滤条件，过滤条件中，可针对不同的表单组件，选择不同的操作符；
如果表单组件的配置做了修改调整，可能会导致原来配置好的条件操作符失效，需要修改后重新发布；
举例：部门组件如果配置的单选模式，条件操作符可选择“等于任意一个”，如果开启了多选，则操作符“等于任意一个”会失效，需要修改为“包含”；

**Q：集成自动化允许自动触发吗？**
集成自动化流程支持设置允许自动触发，开启自动触发后，当其他自动化流或流程满足当前自动化流的触发事件时，当前流会被触发。
**说明**：
自动触发最多支持连续触发三层，即流A => 触发 => 流B，流B => 触发 => 流C，流C => **不能触发** ≠> 流D。
以下情况下，自动触发无法生效：通过集成自动化发起的oa审批，针对oa审批完成后设置的【应用事件触发】不生效，此时手动提交的oa审批可以触发该自动化流。通过集成自动化发起审批，审批对应的流程表单配置的流程【创建】时触发不生效。通过业务关联规则新增、更新数据，目标表单配置的【表单事件触发】不生效。通过集成自动化跨应用触发新增、修改数据，目标表单配置的自动化流不生效。

**Q：为什么变更记录不显示集成删除子表的记录？**
A：集成自动化变更记录暂不支持查看子表数据变更，支持时间请关注后续产品变化。
**Q：通过集成自动化给关联表单组件赋值后，
不会触发关联表单的填充？**
目前通过集成自动化对表单实例进行更新、新增表单实例都不会触发组件的事件，无论是数据联动、公式计算、onChange事件等，同样也不会触发关联表单的数据填充能力。
