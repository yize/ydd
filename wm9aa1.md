# 高级流程节点

高级流程中提供多个是普通流程没有的流程节点，在普通流程的基础上增强了功能，并且可以实现自定义执行人规则和完成策略等功能，有以下这些节点功能：
| **节点名称** | **作用** |
|---|---|
| [**开始节点**](https://www.yuque.com/yida/support/ek8q11) | 是高级流程设计里面发起的节点，流程发起开始操作会伴随着公式校验还是关联操作，节点提交规则的作用是在用户提交流程或者审批人处理流程时通过一些公式校验判断用户是否能执行此操作。 |
| [**人工节点**](https://www.yuque.com/yida/support/ves9o4) | 可根据自身需求自定义审批人规则 |
| [**自动节点**](https://www.yuque.com/yida/support/ctns3l) | 自动节点的主要作用是通过第三方服务选项进行回调，第三方服务需要先进行服务注册，然后即可在第三方服务中选择 |
| [**分支节点**](https://www.yuque.com/yida/support/hhw27w) | 高级流程中支持分支节点的配置，当流程走到分支节点之后，根据分支节点上的规则配置决定流程接下去往哪个分支上走，实现了条件状语，如果……那么……的效果 |
| [**聚合节点**](https://www.yuque.com/yida/support/ee2mei) | 流程中有多条并行的分支，需要多条分支都完成之后才能继续往下走的时候可以使用聚合节点，只有聚合节点所有入线上的节点都执行完，整个流程才会继续执行，实现等待的效果 |
| [**抄送节点**](https://www.yuque.com/yida/support/ghkqiq) | 当审批流程发起后，需要对应的人员查看/知晓该审批单，但是无需审批的场景 |
| [**结束节点**](https://www.yuque.com/yida/support/gkmvl4) | 结束节点代表了整个流程的终点，正确配置的流程应该一步一步最终走到这个节点，代表流程结束 |
